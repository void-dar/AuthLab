What is OAuth 2.0?

OAuth 2.0 is an authorization framework (not authentication!) that lets users grant third-party apps limited access to their account without sharing passwords. Example: "Login with Google" uses OAuth to give an app access to your Google profile.

Roles: Resource Owner (user), Client (the app), Authorization Server (e.g., Google), Resource Server (protected API)

🆚 OAuth 2.0 vs OpenID Connect

OAuth 2.0

  • Handles Authorization
  • "What can this app access?"
  • Issues Access Tokens
  • No standard user identity info
  • Example: GitHub API access

OpenID Connect (OIDC)

  • Handles Authentication
  • "Who is this user?"
  • Issues ID Token + Access Token
  • Standardized user claims (sub, email, name)
  • Example: "Login with Google"
Key insight: OIDC is built on top of OAuth 2.0. It adds an ID Token (a JWT containing user identity claims) to the standard OAuth flow. The openid scope triggers OIDC mode.
