What is Password Hashing?

A cryptographic hash function takes your password and produces a fixed-length fingerprint. The same input always produces the same output, but it's computationally infeasible to reverse — meaning you can't get the password back from the hash.

Modern systems add a salt (random data) before hashing, so two users with the same password get different hashes — defeating rainbow table attacks.

Password strength isn't just about length — it's about entropy: how many bits of randomness exist. A password like correcthorsebatterystaple (4 random words) has more entropy than P@ssw0rd!.

❌ Why MD5 & SHA-256 Are Wrong for Passwords

❌ MD5 / SHA-1 / SHA-256

  • Designed for speed — bad for passwords
  • Billions of hashes per second on GPU
  • Rainbow tables work without salts
  • MD5: completely broken for security

⚠️ bcrypt

  • Designed for passwords — good!
  • Built-in salt — good!
  • Cost factor (work factor) — good!
  • 64-char input limit — limitation
  • Not memory-hard — GPU can attack

✅ Argon2id (Recommended)

  • Winner of Password Hashing Competition
  • Memory-hard (defeats GPU attacks)
  • Configurable: memory, time, parallelism
  • Resistant to side-channel attacks
  • OWASP #1 recommendation (2024)
