What is RBAC?

In RBAC, access is granted based on a user's role (e.g., Admin, Staff, Guest). Permissions are assigned to roles, not directly to users. Users inherit permissions from their roles — simple, predictable, and easy to audit.

📝 Resource Ownership (Posts)

Users can only edit their own posts. Admins can edit any post. This combines RBAC with ownership checks.

💻 Implementation Pattern

Role Check Dependency

@router.get("/admin")
async def admin_only(
    user = Depends(require_role("admin"))
):
    return {"data": "admin only"}

Permission Check

@router.delete("/post/{id}")
async def delete_post(
    user = Depends(
        require_permission("posts:delete")
    )
):
    return {"deleted": True}
