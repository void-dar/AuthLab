Why Go Passwordless?

Passwords are the #1 attack surface: phishing, credential stuffing, weak passwords, database breaches. Passwordless authentication eliminates the password entirely — users authenticate via something they have (device, hardware key) or something they are (biometrics).

✉️ Magic Link Authentication

Generate a one-time login link. In production, this is emailed. Here we simulate it directly.

How Magic Links Work

1
User enters email → server generates a cryptographically random token
2
Token stored in DB with expiration (15 min) + user association
3
Link emailed: https://app.com/auth/magic?token=abc123
4
User clicks link → server validates token → issues session → token deleted (single use)

🔑 Passkeys (FIDO2)

Passkeys replace passwords with public-key cryptography. Your device holds a private key (never leaves your device). The server stores only your public key. Authentication is a cryptographic challenge-response — nothing to steal from the server!

✅ Passkey Advantages

  • Phishing-resistant by design
  • No password to steal from server
  • Biometric-protected on device
  • Works across Apple/Google/Windows
  • Syncs across your devices

⚠️ Considerations

  • Device-dependent (account recovery needed)
  • Requires modern browser/OS
  • Complex server-side implementation
  • Migration from passwords needed

Passkey Registration Flow

1
Server sends a challenge (random nonce)
2
Device creates a key pair for this origin (bound to domain — no phishing!)
3
Device stores private key, sends public key + signed challenge to server
4
Server stores public key. Future logins: sign new challenge with private key → server verifies with public key

🌐 WebAuthn Overview

WebAuthn (Web Authentication API) is the W3C standard that enables browsers to use authenticators (passkeys, hardware keys, biometrics) for authentication.

Standard
W3C + FIDO2
Browser Support
95%+ globally
Algorithm
ES256 (ECDSA)
Phishing
Resistant ✅

WebAuthn Key Concepts

RP (Relying Party): Your web app. Bound to a specific origin (domain). A fake phishing site can't intercept credentials.

Authenticator: The device performing cryptography — could be a hardware key (YubiKey), platform authenticator (Touch ID, Face ID, Windows Hello), or a synced passkey.

Credential: A public/private key pair bound to your RP. Private key never leaves the authenticator.

Real companies using WebAuthn: Google, GitHub, Microsoft, Dropbox, Twitter/X, Facebook — replacing passwords at scale.
